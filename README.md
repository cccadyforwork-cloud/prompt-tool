# 通用产品提示词生成工具 MVP

这是一个本地静态 MVP，用来根据产品资料和模板类型自动生成 Amazon 图组提示词。页面初始状态不展示内置演示参数，需先解析采购单和供应商资料。

## 使用方式

直接用浏览器打开 `index.html`。

功能：
- 输入采购单 PDF、采购单图片、1688 HTML 文件、竞品链接 HTML 文件；1688 和竞品 HTML 可多选。
- 可输入 Amazon 类目模板 Excel，用于按子 SKU 提取采购款式；可用行号范围或 SKU 列表限定要提取的款式。
- 点击解析资料，从资料中提取产品名、款式选项、产品自身数量/套装、规格、材质、结构、适配和卖点线索。
- 选择解析出的当前产品 / 款式。
- 选择参数规格、功能展示、综合卖点 3 个模板之一。
- 图片类型由模板决定，一次性输出完整图组提示词。
- 修改左侧可替换参数，例如 Product Name、Material、Color、Structure、Selling Point。
- 占位符在页面中以蓝色显示，复制时保留纯文本提示词。
- 每张图可单独复制，也可复制当前产品完整图组。

## 数据口径

- 采购单用于确认已采购的产品款式，不把采购下单数量当作产品参数。
- Amazon 类目模板用于确认变体款式：`Parentage Level = Child` 的行数作为采购款式个数；`Variation Theme Name` 中的字段作为款式属性，例如 `SIZE/COLOR` 读取尺码和颜色。父 SKU 只用于归组，不作为采购款式。指定范围支持 `7-17`、`7,9,11-17` 或直接填写 SKU。
- 产品自身数量只从款式名、套装描述或 1688 已确认规格中提取，例如三双装提取为 `3 pairs`。
- 1688 HTML 用于提取产品标题、款式选项、颜色/尺码、材质、规格、尺寸、结构和适配信息。
- 竞品 HTML 用于参考图组逻辑和筛选卖点，不用于补写未确认参数。
- 产品 / 款式名称使用简化后的跨境产品名 + 分类参数，例如颜色或尺码，不展示供应商型号。
- 资料没有明确参数的产品不借鉴、不参照，只写能提取到的规格。
- 提示词中不展示冗长的 verified data 明细，只保留必要占位符。
- 场景展示模板按 7 张图输出：主场景、多场景、多角度、产品说明、卖点图 5、卖点图 6、卖点总览。

## 规则分层

- 全局规则只控制所有图片都必须遵守的底线：Amazon 1:1、清晰度、英文文字、真实产品、不编造参数、不复制竞品、不出现中文源文字。
- 模板规则只控制图组结构：参数规格模板、功能展示模板、场景展示模板各自决定图片数量、主图/多场景/参数/卖点/总结图的构图任务。
- 场景展示模板的主场景图是全局模板规则：必须有吸引力的真实生活方式场景和空间氛围，产品清楚但不能变成鞋/物品特写；需要通过环境、道具、自然光、前后景和留白建立场景吸引力。
- 场景展示模板的多场景图是全局模板规则：必须输出 4 个完整真实使用场景，使用四宫格/四 panel 构图，每个 panel 都要有环境、道具、空间/户外上下文和自然使用动作；不能退化成产品特写、单品摆拍、细节图或选项图。
- 场景展示模板的产品说明图是全局模板规则：必须有清晰主标题，例如 `Product Information` / `Product Details`；只展示 2-3 个已验证信息点，文字要有专业层级、对齐、留白、细分割线/小图标和高级感，不能变成长文、密集表格或无质感的普通标注。
- 场景展示模板和功能展示模板的卖点图是全局模板规则：单张卖点图最多展示 2 个不重复卖点，连续两张卖点图不能重复卖点；必须有一眼可见的大卖点标题，图中文字只使用 3-5 个英文词的短卖点标签，不添加额外口号、长句、重复 claim 或密集文案；场景、道具、动作和环境必须围绕这张图所配对的卖点来证明。
- 类目规则只控制产品语境：通过 `categoryProfile` 识别咖啡滤纸、咖啡金属配件、厨房用品、瑜伽袜/袜类、拖鞋/人字拖/沙滩浴室鞋、雨伞等类目，再提供对应背景、场景和卖点视觉证明。
- 产品跑偏控制分两层：全局规则固定“只生成当前 SKU 的真实产品”，类目规则补充专属结构锁，例如人字拖必须保持源图里的单根 Y 字夹脚带、一个中央夹脚柱、两个侧锚点和开放后跟，禁止被改成凉拖、宽带拖、双带拖、后跟带或脚踝带结构。
- 新增或优化某类产品时，优先新增/修改类目规则，不把类目场景硬写进全局规则或模板规则，避免换产品后串场景。
